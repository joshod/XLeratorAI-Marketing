Architecture

Three independent layers. One stateless application.

Every XLeratorAI accelerator separates concerns into three independently configurable layers. The application itself is stateless — the same secure codebase regardless of how you deploy it. This means your organization can change its AI provider or data strategy without rebuilding the product.

  • Application Layer: A single, stateless accelerator. No data persists in the application itself — it processes, delivers, and moves on.
  • AI / LLM Layer: Choose where inference runs — cloud API, your enterprise tenant, or fully on-premises. Same functionality, different trust boundaries.
  • Data / Storage Layer: Choose what persists and where — from zero retention to a fully structured data warehouse under your control.

AI / LLM Layer

Choose how AI processes your data.

The AI layer handles all inference and generation. Every option delivers the same accelerator functionality — the difference is where the model runs and who manages it.

☁️
Default

Cloud API

Managed AI services operated by XLeratorAI. Fastest to deploy, lowest onboarding friction. Ideal for organizations comfortable with cloud-based AI services and standard data processing agreements.

  • Fastest deployment — days, not weeks
  • Managed and maintained by XLeratorAI
  • Standard data processing agreement (DPA)
  • Lowest upfront investment
🏢
Enterprise

Enterprise AI Platform

Microsoft Azure AI Foundry or equivalent enterprise platforms. AI runs within your cloud tenant — data never leaves your security boundary. Ideal for organizations with existing enterprise cloud agreements and data residency requirements.

  • Data stays within your cloud tenant
  • Leverages existing enterprise agreements
  • Meets data residency and sovereignty requirements
  • Client IT team manages model access and policies
🔒
Air-Gapped

Local / Private Models

Ollama or similar local model runners. Complete air-gapped operation — zero data leaves the network. Ideal for defense, regulated industries, or organizations with strict zero-trust policies.

  • Complete air-gapped operation
  • Zero data leaves the network
  • Full client control over model selection and updates
  • Supports zero-trust and classified environments
Data / Storage Layer

Choose what persists — and where.

The data layer is completely independent from the AI layer. Whether you use cloud AI or local models, you have full control over what data is stored, where it lives, and who has access.

⚡

Stateless Operation

Data is processed in-memory only. Nothing persists after the session ends. Ideal for initial evaluations, sensitive ad-hoc analysis, or organizations that prohibit data retention entirely.

  • Zero data retention
  • Session-scoped processing only
  • No database provisioning required
  • Simplest compliance posture
🗃️

Client-Managed Storage

SQLite on client infrastructure or cloud object storage such as AWS S3 or Azure Blob. Your organization owns, controls, and has full visibility into all stored data — no third-party custody.

  • Client owns and controls all data
  • SQLite, AWS S3, or Azure Blob options
  • Full audit trail and access logging
  • Encryption at rest and in transit
📊

Dedicated Data Warehouse

XLeratorAI provides the schema and data model; your organization provisions the database. Enables structured analytics, historical trending, and cross-accelerator reporting for portfolio-level intelligence.

  • XLeratorAI-designed schema and data model
  • Client-provisioned database infrastructure
  • Cross-accelerator analytics and reporting
  • Historical trending and benchmarking
